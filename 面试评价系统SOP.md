## 一、项目概述

这是一个**面试评价数据化 + 人才资产沉淀**的轻量级系统。核心理念是把面试官脑中的模糊判断，变成结构化的、可分析、可回溯的数据。一站式面试管理工具，覆盖从简历导入、流程追踪、面试评分到人才库归档的完整闭环。

**技术栈**：纯前端 HTML + JavaScript + ECharts 图表，可选后端（Node.js/Python）支持多人协作和 PDF 解析。

**运行方式**：
- **单人使用**：双击 `index.html`，数据存浏览器 LocalStorage
- **多人共享**：双击 `启动服务器.bat`（或 `node server.js`），数据存 `data/shared_data.json`，局域网可访问
- **云端访问**：`https://hilarious-souffle-3038d3.netlify.app`

## 二、系统现状速览

### 模块结构

三个主Tab：**候选人**（核心工作区）、**数据仪表盘**（统计可视化）、**能力模型**（后台配置）。

#### 候选人 Tab（三合一，2026-06-22）

候选人管理、面试评价、人才库合并为同一个界面。

- **列表页**（全屏）：展示所有候选人，支持筛选（岗位/阶段/评级/职级/关键词/年月）、排序、批量删除。每人一张扁平卡片，右侧醒目状态标签——待评价状态带黄色脉冲动画，特殊情况带⚠️橙色原因文字。
- **详情页**（全屏）：点击候选人进入，三个子tab：
  - **简历 & 进度**：流程追踪（初试→复试→终面圆点+箭头）、简历（PDF嵌入或文本）、快捷操作按钮
  - **面试评价**：完整评分表，先定级再评分（见下文）
  - **历史评价**：该候选人所有轮次记录，可加载到评价表复用

导入方式：粘贴文本、选择文件（PDF/TXT，自动调用 pdftotext 提取）、快速添加。导入时自动识别姓名（百家姓校验+黑名单过滤）、手机号（11位校验）、邮箱、公司（200+游戏公司库），并检测重复。

#### 面试评价流程（2026-06-22）

采用**先定级、后评分**的两步流程：

1. **Step 1：选择建议职级** — Expert（对标网易P5）/ Senior3（P4）/ Senior2（P3）/ Senior1（P2）
2. **Step 2：系统自动匹配评价模板** — 不同职级对应不同的硬性能力维度

**职级模板逻辑**：13个岗位 × 4个职级 = 52套独立模板。同一岗位不同职级考察维度完全不同：

| 职级 | 侧重方向 |
|------|----------|
| Expert | 架构设计、团队管线、方法论输出、行业影响力 |
| Senior3 | 复杂模块设计、跨部门协作、引擎深度使用 |
| Senior2 | 独立完成任务、基础协作、执行力 |
| Senior1 | 基础理解、工具学习、游戏热情、主动性 |

软性素质（沟通、驱动力、稳定性、AI能力、职业规划、意愿度）各职级通用，但硬性/软性权重随职级浮动（Expert 硬性占65%~70%，Senior1 占50%~55%）。

#### 流程管道

8个阶段，自动流转：
```
待初试 → 初试待评 → 待复试 → 复试待评 → 待终面 → 终面待评 → 已发Offer
                                                              ↘ 特殊情况
```

- 点击"已初试/已复试/已终面"按钮推进阶段
- 在"待评"状态点击"评价"→ 自动填入候选人信息 → 评分保存后自动进入下一阶段
- 特殊情况可标记原因（放弃面试、已有offer等），橙色高亮

### 数据仪表盘

12张可视化图表：阶段通过率饼图、维度均分柱状图、面试官打分倾向雷达图、月度趋势、转化漏斗、评分分布直方图、岗位×维度热力图、面试官对比表、箱线图、招聘周期、公司来源分布、数据总览表。每个图表下方自动生成分析结论文字。

## 四、技术决策记录

| 决策 | 选择 | 原因 |
|------|------|------|
| 前端框架 | 原生 HTML + JS | 0 依赖，双击即用，HR 的电脑不需要装任何东西 |
| 图表库 | ECharts（CDN） | 国内镜像稳定，功能全面，支持所有图表类型 |
| 数据存储 | LocalStorage + JSON 文件 | 轻量，不需要数据库，服务端模式下可共享 |
| 服务端方案 | Node.js（最终） | 比 PowerShell 稳定，比 Python 部署简单 |
| 评分方式 | 1-5 星点击 | 减少主观偏差，强制逐维度打分 |
| 能力模型 | JSON 配置，可编辑 | 不需要改代码就能调整评分标准 |
| Tab 架构 | v2.0 合并为3个 | 5个Tab流程割裂，合并后更流畅 |
| 职级模板 | 岗位×职级二维矩阵 | 不同级别考察重点不同，不能用同一套标准 |
| 密码管理 | LocalStorage + 密保重置 | 不需要后端，安全够用 |
| 部署方式 | 双击 HTML 或 Node.js 服务器 | file:// 下功能完整，服务器模式下可多人协作 |

## 九、常用操作速查

### 启动系统

```powershell
# 方式1：双击 HTML（单人使用，数据存本地）
D:\绵羊札记\面试评价平台\index.html

# 方式2：Node.js 服务器（多人共享，数据存 shared_data.json）
双击 D:\绵羊札记\面试评价平台\启动服务器.bat
# 或
node D:\绵羊札记\面试评价平台\server.js
```

### 导入简历

- **粘贴导入**：点击「📋 粘贴导入」→ 粘贴简历文本 → 确认
- **文件导入**：点击「📂 选择文件导入」→ 选择 PDF/TXT 文件
- **批量 PDF 转文本**：拖拽文件夹到 `PDF转TXT-拖拽文件夹到这里.bat`

### 配置能力模型

1. 切换到「⚙️ 能力模型」Tab
2. 点击「🔓 管理员编辑」→ 输入密码登录
3. 编辑/新增岗位模型
4. 保存后所有下拉框自动更新

### 恢复密码

- 点击登录弹窗中的「忘记密码？」
- 回答密保问题（你第一份工作的公司名称是？）
- 设置新密码
