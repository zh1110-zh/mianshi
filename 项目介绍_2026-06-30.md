## 一、项目背景与目标

游戏招聘中，面试官的评价往往停留在口头反馈或简单的"通过/不通过"，面试数据无法沉淀、面试质量无法衡量。市面上没有一款轻量、灵活、适合中小企业游戏团队使用的面试评价工具。

目标：**把面试官脑中的模糊判断，变成结构化的、可分析、可回溯的数据**。不依赖第三方付费平台，自己就能维护。

## 二、系统现状：一站式面试管理闭环

打开一个网页，完成从简历导入到人才库归档的完整流程：

### 2.1 简历导入（智能解析）

- 支持 PDF/TXT 拖入导入、文本粘贴导入、手动快速添加三种方式
- 自动解析：姓名（百家姓校验 + 80个黑名单过滤）、11位手机号（三策略提取：标准格式→全数字搜索→去空白全局搜）、邮箱、200+游戏公司识别
- 重复检测：按姓名/手机号/邮箱三重匹配，避免重复录入

### 2.2 流程管道

专业面试第一轮 → 专业面试第二轮 → 制作人面试 → 拟offer

- 每个阶段有"待面试"和"待评价"两个状态，流程自动推进
- 制作人面试完成后，只有整体评价达到"超越预期"或"达到预期"，系统才标注"拟offer"
- 特殊情况可手动标记原因（放弃面试、已有offer等），卡片上直接显示橙色原因文字
- 卡片上每轮显示分数 + ✓通过 / ✗未通过标签，进度一目了然

### 2.3 结构化评价（核心创新）

**先定级、再评分**——这是整个系统最大的设计亮点。

传统做法是所有候选人用同一套标准打分，但 Expert 和 Senior1 考察的维度完全不同。现在的流程：

1. Step 1：面试官选择建议职级（Expert/Senior3/Senior2/Senior1）
2. Step 2：系统自动匹配对应该岗位+该职级的评价模板
3. 面试官按模板逐项评分（1-5星）+ 必须写文字备注
4. 综合评分自动计算（硬性能力 × 权重 + 软性素质 × 权重）

**13个岗位 × 4个职级 = 52套独立评价模板**。举例说明不同职级考察维度的差异：

| 职级 | 关卡策划考察重点 |
|------|----------|
| Expert | 关卡架构与世界观整合、大型团队管线搭建、技术美术边界理解、方法论建设与知识传承 |
| Senior3 | 复杂关卡设计与迭代、跨部门协作落地、UE4/5深度使用、性能优化与资源管理 |
| Senior2 | 独立关卡设计能力、Unity/UE引擎使用、基础协作与沟通、游戏理解与设计迭代 |
| Senior1 | 基础关卡设计理解、工具学习与执行力、逻辑思维与文档、游戏热情与主动性 |

软性素质6项（沟通能力&性格、驱动力、稳定性、AI能力、职业规划清晰度、意愿度）各职级通用，但硬性/软性权重随职级浮动（Expert 硬性占65%-70%，Senior1 占50%-55%）。

### 2.4 数据仪表盘（12张图表）

所有图表纯 Canvas 手绘，不依赖第三方图表库：

1. **阶段通过率饼图**：一面/二面/制作人面试/不通过分布
2. **各维度平均分柱状图**：识别最强项和最弱项
3. **面试官打分倾向雷达图**：发现谁偏严谁偏松
4. **月度面试量 & 通过率趋势图**：双轴柱状+折线
5. **转化漏斗**：简历库→一面→二面→制作人→通过
6. **评分分布直方图**：检测评分是否通胀（8个区间）
7. **岗位×维度热力图**：各岗位在各能力维度的均分矩阵
8. **面试官对比表**：场次/平均分/通过率
9. **箱线图**：面试官评分分布（min/Q1/中位数/Q3/max）
10. **招聘周期图**：各岗位从初面到终面的平均天数
11. **公司来源分布**：候选人来源公司 Top15
12. **数据总览表**：按岗位汇总面试数/通过率/均分/评级分布

每张图下方自动生成分析结论文字（如"通过率从1月的60%上升至3月的75%，呈上升趋势"）。

### 2.5 候选人才库

- 所有候选人统一列表，支持按岗位/阶段/面试结果/职级/关键词/年月筛选
- 排序：最近添加、年限、姓名、综合分、面试日期
- 批量操作：全选 + 批量删除
- 每张卡片展示：姓名、岗位、年限、公司、流程进度、评分、通过/未通过状态
- 待评价阶段有黄色脉冲动画标签，特殊情况直接显示原因

### 2.6 能力模型配置（管理员功能）

- 密码保护 + 密码修改 + 密保找回
- 可新增/编辑/删除岗位的能力维度、权重、硬软占比
- 当前已配置 13 个岗位（数值策划、系统策划、战斗策划、关卡策划、3D角色、角色原画、任务策划、3D动作、UI动效、客户端开发、引擎工程师、APM、UI策划）

## 三、近三周迭代历程

### 第一周（6.11-6.14）：基础可用

- 5个独立Tab：候选人管理 / 面试评价 / 数据仪表盘 / 人才库 / 能力模型
- 13个岗位的固定评价维度
- 流程管道：待初试→初试待评→待复试→复试待评→待终面→终面待评→已发Offer
- 4张基础图表，浏览器 LocalStorage 存储
- 支持粘贴导入和快速添加候选人
- 服务端支持（Node.js/Python/PowerShell三套方案），局域网共享数据
- PDF 简历解析（pdftotext + pdf.js 双方案）

### 第二周（6.22-6.24）：架构重构

- **三Tab合一**：候选人管理+面试评价+人才库合并为"候选人才库"统一Tab，全屏列表→全屏详情，去掉了子Tab导航
- **先定级再评分**：选职级后系统自动匹配52套模板，替代原来固定维度
- **UI全面重设计**：紫色渐变Header、卡片阴影+圆角+hover上浮、毛玻璃弹窗、表单focus光圈
- **状态标签突出**：待评价状态黄色脉冲动画，特殊情况橙色原因直接显示
- **Supabase云同步**：数据存云数据库，本地和云端共享
- **GitHub + Netlify部署**：代码上云，换电脑也能改，自动部署
- 职级对标：Expert(P5)/Senior3(P4)/Senior2(P3)/Senior1(P2)

### 第三周（6.25-6.30）：打磨优化

- **阶段名称统一**：初试→专业面试第一轮，复试→专业面试第二轮，终面→制作人面试
- **卡片评分可视化**：每轮显示分数 + ✓通过 / ✗未通过标签
- **流程逻辑优化**：只有制作人面试达到预期以上才显示"拟offer"，低于预期不自动进特殊情况
- **电话识别升级**：三策略提取（标准格式→全数字搜索→去空白全局搜），覆盖PDF换行拆散
- **评价按钮直达**：点击评价按钮直接进入评价页，表单自动填入候选人全部信息
- **简历展示优化**：取消高度限制、字号14px、PDF嵌入500px
- **筛选栏优化**：全部评级→面试结果，阶段下拉改为专业面试通过/未通过+拟offer
- **GitHub自动推送**：改完代码自动推送到 `github.com/zh1110-zh/mianshi`
- **三份文档同步**：系统SOP + 变更记录 + 版本备份，存于绵羊札记

## 五、技术亮点

- **零依赖**：纯 HTML+CSS+JS，不依赖 React/Vue 等框架，不依赖 ECharts 等图表库
- **12张Canvas手绘图表**：自绘 HiDPI 适配、颜色主题、分析结论自动生成
- **多策略信息提取**：姓名识别用了百家姓校验+黑名单过滤+四重策略，手机号三策略提取
- **单文件架构**：一个 index.html 包含全部功能，部署就是复制一个文件
- **PDF解析双方案**：服务端 pdftotext + 浏览器端 pdf.js，随时fallback
- **数据三层存储**：服务端 JSON + Supabase 云数据库 + LocalStorage 浏览器备份
